# はじめに

Tracewright Narrative Uncertainty Map は、文章や資料を「AIか人間か」で判定するためのものではありません。

論文、草稿、創作、メール、公開発言、来歴資料、複数の資料セットなどを、自分のAIと一緒に丁寧に読み直し、根拠、矛盾、不確実性、代替説明、次の確認事項を整理するためのレビュー補助ツールです。

## まずは一番簡単な使い方

ChatGPT、Claude、Gemini など、普段使っているAIに次の1ファイルを添付してください。

```text
USE_THIS_WITH_YOUR_AI.ja.md
```

そして、チャット画面にこの文章を貼って送ってください。

```text
添付したTracewrightの説明ファイルを先に読んでください。

まだ分析したい資料はアップロードしません。
まず、このAI環境に資料を入れてよいか確認するための注意点を教えてください。
そのあと、どんな種類のレビューをしたいのか、私に質問しながら手順を案内してください。

AI判定器のような結論ではなく、根拠・矛盾・不確実性・代替説明・次の確認事項が見えるレビュー地図を作りたいです。
```

あとはAIの質問に答えてください。

最初から大量の資料を入れず、短い抜粋や少数のファイルから試すのがおすすめです。

迷ったら、他のファイルはまだ開かず、まず `USE_THIS_WITH_YOUR_AI.ja.md` だけをAIに添付してください。

## ダッシュボードを作りたい場合

Codexなどを使って、HTMLダッシュボードやJSONデータとして整理したい場合は、英語版の次のファイルを開いてください。

```text
DASHBOARD_BUILDER_GUIDE.md
```

デモ用ダッシュボードはここにあります。

```text
dashboard/index.html
```

## アップロード前の確認

AIに資料を渡す前に、必ず確認してください。

- その資料は、個人情報、機密情報、未公開原稿、法的に慎重な情報、第三者の資料ではありませんか？
- そのAIサービスが、アップロードしたファイルをどう扱うか理解していますか？
- そのAI環境に入れてよい資料ですか？
- 自分にその資料をアップロードする権利がありますか？

少しでも不安がある場合は、匿名化、要約、抜粋、架空資料、公開済み資料で試してください。

Tracewright自体が資料を守ってくれるわけではありません。プライバシーや安全性は、あなたが使うAIサービス、アカウント設定、ローカル環境、アップロードする資料の選び方に依存します。

雇用、法務、学術評価、評判、来歴、出版、金銭的判断に影響するレビューでは、Tracewrightの出力は人間専門家レビューの準備として扱ってください。
